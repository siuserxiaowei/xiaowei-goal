# Business Priority

Use this reference before finalizing a business, product, content, SEO, growth,
or competitor goal. The point is to decide whether the work deserves a full
goal, a small experiment, a pause, or a rejection.

## Priority Dimensions

Score qualitatively:

```text
业务价值：高 / 中 / 低
证据强度：高 / 中 / 低
执行成本：高 / 中 / 低
分发潜力：高 / 中 / 低
变现路径：清晰 / 不清晰
风险：高 / 中 / 低
建议：立即做 / 先小实验验证 / 暂缓 / 不建议做
```

## Decision Rules

- High value + medium/high evidence + manageable cost: `立即做`.
- Unclear value or evidence but low cost: `先小实验验证`.
- Low value + weak evidence + medium/high cost: `暂缓`.
- High legal/privacy/compliance/copyright risk without authorization: `不建议做` or pause.
- If the user asks to build a large system before validating demand, recommend a smaller validation loop first.

## Priority Field

Generated goals should include:

```text
优先级判断：业务价值=...；证据强度=...；执行成本=...；分发潜力=...；变现路径=...；风险=...；建议=...
```

If the recommendation is `暂缓` or `不建议做`, the goal should not pretend the
main path is full implementation. It should propose reframing or a small
validation experiment.

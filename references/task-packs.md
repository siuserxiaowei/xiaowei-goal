# Task Packs

Use this reference to make generated `/goal` commands more directly executable.
Choose the smallest task pack that matches the user's business goal.

## App MVP Research Pack

Use for mobile apps, web apps, SaaS tools, Chrome extensions, AI tools, and product MVPs.

Require mapping research to:

- target user and job-to-be-done
- first-run experience
- core workflow
- activation moment
- retention or habit loop
- MVP feature list
- pricing or monetization hypothesis
- trust, privacy, support, and review risks
- next implementation `/goal`

## Website / Landing Page Revamp Pack

Use for homepages, product pages, landing pages, waitlists, pricing pages, and lead-generation pages.

Require mapping research to:

- first viewport message
- navigation
- page section order
- CTA path
- proof points
- pricing or lead capture
- objection handling
- SEO entry points
- analytics or experiment plan
- next implementation `/goal`

## SEO Content Cluster Pack

Use for SEO plans, content sites, topic clusters, programmatic pages, and comparison pages.

Require mapping research to:

- seed topics and search intent
- keyword cluster or topic cluster
- SERP competitors
- page types
- internal linking plan
- content brief for priority pages
- freshness and authority requirements
- conversion path from content to product/business action
- next content production `/goal`

## Competitor Analysis Pack

Use for direct competitors, adjacent products, category mapping, and strategic comparison.

Require mapping research to:

- direct and adjacent competitor list
- positioning and target audience
- pricing and packaging
- feature and workflow comparison
- acquisition channels
- trust signals
- user complaints and objections
- what can be borrowed safely
- what must not be copied
- next business or implementation `/goal`

## Growth Experiment Pack

Use for acquisition, activation, retention, referral, conversion, lifecycle, and monetization experiments.

Require mapping research to:

- target segment
- growth channel
- user trigger or pain
- experiment hypothesis
- offer, hook, or message
- funnel step
- success metric and guardrail metric
- sample size or minimum evidence note when useful
- effort, risk, and priority
- next experiment execution `/goal`

## Task Pack Field

Generated research goals should include one compact `任务包` field:

```text
任务包：[App MVP 研究包 / 网站/落地页改版包 / SEO 内容集群包 / 竞品分析包 / 增长实验包]；本次必须把研究结论映射到[pack-specific outputs]。
```

If more than one pack applies, choose the primary pack and mention the secondary pack in `可选调整`.

## Domain Packs

After choosing the general task pack, read `domain-packs.md` when the task
matches one of these domains:

- AI tool site
- Chrome extension
- global/outbound SaaS
- Xiaohongshu/Douyin validation
- GitHub open-source growth

The domain pack should make the general task pack more specific, not replace it.

# AI 教育产品规划与交流：xiaowei-goal 拆解

资料范围：飞书智能纪要《AI教育产品规划与交流》，会议时间 2026-05-10 19:56-20:49。本文只发布结构化理解、外部资料补充和业务拆解，不公开原始逐字稿与私有飞书链接。

处理方式：按 `xiaowei-goal` 三阶段流程执行：先做广域联网调研，再做 Deep Research 交叉判断，最后落到业务动作，并用「道法术器势」拆解。

## 一句话结论

这份纪要的核心是：**AI 教育产品不能只靠“更强模型”和“更快开发”取胜，真正要打的是老师的课堂采用成本、学校合规边界、班级激励效果、平台获客稳定性和品牌资产。** 当前更合理的策略是先做国内 K12 教师场景，把一个班级管理小工具打磨到高转化、高留存，再考虑海外和平台化。

## 逐句级理解

| 纪要信息点 | 我的理解 | 业务含义 |
|---|---|---|
| “两个人团队，创业者加合伙人，自由职业者协助。” | 团队极轻，优点是快，短板是运营、合规、客服、稳定性。 | 早期只能做窄场景，不要做复杂平台。 |
| “看到 AI 生成网站卖得不错。” | 这是市场信号，但不是充分证据。 | 要继续验证购买者是谁、为什么买、是否复购。 |
| “用更强 ChatGPT/Gemini 是降维打击。” | 技术优势存在，但窗口很短，别人也能用同样工具。 | 真壁垒要转向用户理解、场景细节、品牌和数据。 |
| “加入宠物概念，属于原创设计。” | 宠物是情绪化和差异化入口。 | 要把宠物变成课堂行为激励机制，而不是装饰。 |
| “产品是纯网站，不是小程序。” | 网站利于快速迭代和跨设备访问，但老师使用习惯可能弱。 | 必须做桌面快捷方式、免登录体验、二维码入口和超短教程。 |
| “目标是国内 K12 老师。” | 用户清晰，但购买者、使用者、受影响者可能不同：老师、学生、家长、学校。 | 页面和功能要分别回应老师效率、学生参与、学校合规。 |
| “网站可添加桌面快捷方式。” | 这是降低使用门槛的技巧。 | onboarding 第一屏就应引导“添加到桌面”。 |
| “徽章、升级效果、声音。” | 属于游戏化反馈机制。 | 每个反馈都要对应课堂行为目标，如纪律、作业、发言、互助。 |
| “班级积分 + 宠物。” | 积分是行为记录，宠物是情绪激励。 | 关键是积分规则是否方便老师设置，是否会伤害学生自尊。 |
| “老师不熟悉网站，需要培训。” | 教师采用技术的最大障碍之一是复杂度和自我效能。 | 设计原则：不用学习也能完成第一堂课使用。 |
| “教育局有约束。” | K12 场景不是普通消费软件，涉及未成年人、学校管理、数据和价值导向。 | 避免排名羞辱、隐私暴露、过度游戏化、诱导消费。 |
| “排行榜可能不允许。” | 公开竞争可能带来压力和合规问题。 | 改成小组协作、个人成长、老师可见、匿名或非公开反馈。 |
| “小红书压制虚拟商品，产品下架、账号降权、流量下降。” | 平台规则和风控会直接影响收入。 | 小红书只能做获客入口，不能作为唯一销售和交付系统。 |
| “商标被抢。” | 品牌保护滞后会造成长期损失。 | 立刻查商标、注册域名、统一品牌资产。 |
| “更换或增加品牌词。” | 这是短期止损，不是长期壁垒。 | 新品牌必须同步做商标、域名、账号和官网。 |
| “先做国内，暂不出海。” | 资源有限时聚焦是正确选择。 | 海外要等国内产品留存、付费和案例稳定后再做。 |
| “海外存在英文、推广、文化障碍。” | 出海不是翻译界面，而是重做渠道和语境。 | 后续可先用英文 landing page 测需求，不急着开发海外版。 |
| “外包能外包的工作，创始人做决策方向管理。” | 创始人时间要放在关键判断上。 | 外包内容、素材、客服、测试；核心需求、产品、数据复盘不能外包。 |
| “国内用小红书和卡密销售。” | 卡密适合早期快速变现，但不适合长期订阅和用户资产沉淀。 | 过渡到官网账号体系、支付、订阅和使用数据。 |
| “香港服务器，后续再备案。” | 这是早期速度选择，但面向国内用户长期仍要考虑备案、访问速度和支付。 | 如果国内长期经营，尽早规划 ICP、域名、隐私政策、用户协议。 |
| “用 Claude/Codex/Gemini 开发。” | 多模型协作能提高开发效率。 | 建立统一需求文档、测试清单、发布流程，避免 AI 改乱代码。 |
| “没有接支付，卡密激活。” | 先卖起来是好事，但自动化和留存数据不足。 | 下一步应接入账号、激活、到期、续费、退款和客服流程。 |
| “学习刘小白课程，边做边学。” | 学习方式是项目驱动。 | 把每次学习沉淀为自己的 SOP，而不是只看课。 |
| “曝光是杠杆，曝光后资源会来。” | 内容输出能带来用户、合作、人才和信任。 | 创始人可以做 AI 教育产品实战账号，但内容要绑定产品进展。 |
| “社群/线下活动能链接优秀的人。” | 资源密度会影响机会。 | 有选择地参加教育、AI、出海、产品社区，带着具体问题去。 |
| “咖啡因、惠州生活、旅行社交。” | 这些不是产品核心，但反映创业者状态和生活方式。 | 创始人要管理能量、节奏和社交，不要把执行力消耗在低价值分散上。 |

## 阶段 2：Deep Research 判断

### 已被外部资料强化的判断

1. **教育数字化确实是大趋势。** 国家中小学智慧教育平台的大规模使用说明老师和学生已经处在数字化教学环境里，轻量工具有机会作为补充存在。
2. **游戏化有效，但必须服务教育目标。** Frontiers 的系统综述支持游戏化对参与和动机的价值，但也强调设计目的、复杂度和教育需求匹配。
3. **教师采用成本是核心问题。** 教师是否觉得有用、是否容易用、是否有培训和机构支持，会直接影响工具能否进入课堂。
4. **国内 AI 教育产品要提前考虑合规。** 网信办生成式 AI、内容标识、备案和未成年人场景要求，都决定了产品不能只看功能。

### 需要修正的判断

1. **“更强模型”不是长期壁垒。** 当竞品也能使用强模型时，壁垒会转移到场景理解、课堂规则、品牌、渠道、数据和用户关系。
2. **“排行榜”要慎用。** 对 K12 学生公开排名可能带来压力、比较和管理风险，建议优先做私密成长、小组协作、老师可控反馈。
3. **“纯网站”需要用产品设计弥补习惯劣势。** 如果老师习惯小程序/APP，网站必须做到极低登录摩擦和极短学习成本。

### 仍需验证的问题

1. 老师买单的真实原因：好玩、提高课堂纪律、节省时间、学生喜欢，还是可展示成果。
2. 老师是否愿意从 9.9 元一次性购买转为月费/年费。
3. 学校是否允许在课堂公开展示宠物、积分、音效、排名或学生昵称。
4. 小红书当前虚拟商品/教育资料类目规则，需要在商家后台或官方学习中心二次核验。

## 阶段 3：应用到业务

| 业务环节 | 可执行动作 | 验证指标 |
|---|---|---|
| 产品定位 | 定义为“老师 3 分钟可上手的班级激励工具”，不是完整教学平台。 | 老师首次使用时间、第一堂课完成率。 |
| 核心功能 | 班级创建、学生/小组、积分规则、宠物成长、老师可控展示、奖励兑换。 | 日活班级数、每班每周积分次数、老师复访。 |
| 游戏化设计 | 徽章、升级、音效、宠物反馈必须绑定具体课堂行为。 | 学生参与度、老师主观满意度、课堂干扰投诉。 |
| 合规设计 | 默认不公开真实姓名；排行榜可关闭；AI 生成内容加标识；隐私政策和用户协议。 | 投诉率、审核通过率、学校/老师接受度。 |
| 获客渠道 | 小红书做案例和场景内容，官网承接注册，社群做老师反馈。 | 笔记咨询率、官网注册率、社群活跃、转化率。 |
| 支付订阅 | 卡密作为过渡；逐步转账号体系、月费、年费、班级套餐。 | 首购率、续费率、退款率、客服成本。 |
| 品牌资产 | 商标检索、域名、账号、LOGO、产品名、案例页同步建立。 | 搜索品牌词、投诉/盗版处理效率。 |
| 出海准备 | 国内验证稳定后，先做英文 landing page 和竞品调研，不急开发。 | 英文页访问、留资、搜索量、竞品价格。 |

## 道法术器势拆解

| 层级 | 拆解 | 对应动作 |
|---|---|---|
| 道 | 教育产品的本质不是炫技，而是降低老师负担、改善课堂参与，并守住未成年人和学校场景边界。 | 所有功能都要回答：老师更省事了吗，学生更投入了吗，学校能接受吗。 |
| 法 | 聚焦国内 K12 老师，先把一个轻量班级激励工具打透，再扩展支付、订阅、案例、出海。 | 先做 30 个真实老师访谈/试用，再决定路线。 |
| 术 | 小红书内容、桌面快捷方式、二维码入班、宠物成长、积分规则、匿名/小组反馈、官网承接、卡密过渡。 | 用 14 天跑通“内容 - 注册 - 首次使用 - 复访 - 付费”闭环。 |
| 器 | Claude/Codex/Gemini、官网、账号系统、支付、数据分析、知识库、隐私政策、AI 标识、商标/域名。 | 建立最小产品运营后台和数据看板。 |
| 势 | 教育数字化、AI 工具普及、内容平台电商、K12 合规约束、AI 内容标识同步增强。 | 顺势做合规轻工具，不逆势做重平台和高风险公开排名。 |

## 下一步 `/goal`

```text
/goal 围绕“国内 K12 老师班级激励宠物产品”执行三阶段研究工作流。
阶段 1 - 广域调研：检索 25 个来源，覆盖小红书老师内容、竞品班级管理/积分/游戏化工具、国家教育数字化资料、游戏化教育研究、教师技术采用研究、AI 教育合规资料、官网/APP 获客资料。
阶段 2 - Deep Research：筛选 10 个高价值来源深读，提取老师真实痛点、课堂使用限制、竞品功能、付费方式、获客内容、合规风险和证据强度。
阶段 3 - 业务应用：输出产品定位、MVP 功能、首屏文案、官网结构、小红书 20 个选题、老师试用脚本、订阅方案、合规清单和 14 天验证计划。
验证方式：每个核心功能必须对应老师痛点；每个增长动作必须对应渠道指标；事实、推断、建议分开写。
限制：不公开学生隐私；不默认使用公开排名；不复制竞品素材；AI 生成内容必须标识和人工审核。
```
